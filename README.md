# ワンデーアキュビューオアシス 最安値ランキングサイト

楽天市場・Yahoo!ショッピングの「ワンデーアキュビューオアシス
90枚入り×2箱セット（180枚）」の価格を毎日自動取得し、
安い順トップ5・1枚あたり単価を表示する静的サイトです。

- 価格取得: 楽天商品検索API / Yahoo!ショッピング商品検索API（公式API）
- 比較単位: 90枚入り×2箱セット（180枚）に固定
  （1箱・3箱等の商品は商品名フィルタで除外し、単価が混在しないようにしています）
- アフィリエイト: もしもアフィリエイト（楽天）／バリューコマース（Yahoo!）に対応
- 自動更新・公開: **すべてGitHubだけで完結**します。
  GitHub Actionsが毎日 日本時間 朝6時に価格を取得してリポジトリにコミットし、
  GitHub Pagesがその内容を自動で公開します（サーバーの契約やFTP設定は不要です）
- 独自ドメイン: お持ちのドメイン（のサブドメイン）をGitHub Pagesに割り当てて公開できます

構成:
```
docs/                 ← GitHub Pagesが公開するフォルダ（サイト本体）
  index.html           サイト本体（data.jsonを読み込んで表示するだけ）
  style.css            デザイン
  data.json            価格データ（毎日自動で上書きされる）
  CNAME                独自ドメインの設定ファイル（ご自身のドメインに書き換えてください）
scripts/
  fetch-prices.mjs      楽天/Yahoo APIを叩いてdata.jsonを作るスクリプト
.github/workflows/
  update-prices.yml     毎日自動実行 → data.json更新 → GitHubにpush（Pagesが自動反映）
```

## セットアップ手順

### 1. このプロジェクトをGitHubリポジトリにする

コマンドラインを使わない場合:
1. [github.com](https://github.com/) で無料アカウントを作成（お持ちでなければ）
2. 右上「+」→「New repository」→ 名前を決めて「Create repository」
   - Public（公開）リポジトリにしてください（Privateだと有料プラン以外ではGitHub Pagesが使えない場合があります）
3. 作成直後の画面の「uploading an existing file」をクリック
4. このzipを解凍し、`lens-price-site` フォルダの中身一式
   （`.github`、`docs`、`scripts`、`README.md`、`package.json`）をドラッグ&ドロップ
   （`.github` のような「.」始まりのフォルダもフォルダごとドラッグすればアップロードされます。Chrome/Edge推奨）
5. 「Commit changes」

コマンドラインを使う場合:
```bash
cd lens-price-site
git init
git add .
git commit -m "init: lens price comparison site"
gh repo create <お好きなリポジトリ名> --public --source=. --push
```

### 2. GitHub Secretsを登録する

**Settings → Secrets and variables → Actions → New repository secret**
で以下を登録します。

| Name | 値 | 必須 |
|---|---|---|
| `RAKUTEN_APP_ID` | お持ちの楽天ウェブサービスのApplication ID | ○必須 |
| `YAHOO_CLIENT_ID` | Yahoo!のClient ID | 任意（後日追加可） |

（アフィリエイト用のSecretsは、下記「6. アフィリエイト収益化」を参照）

### 3. GitHub Pagesを有効にする

1. リポジトリの「Settings」→ 左メニュー「Pages」
2. 「Build and deployment」の Source を **Deploy from a branch** に
3. Branch: `main`、フォルダ: **`/docs`** を選択→「Save」
4. 数分後、`https://ユーザー名.github.io/リポジトリ名/` が公開されます（まずはこのURLで動作確認できます）

### 4. 独自ドメインを設定する

**4-1. ドメイン管理画面でDNSレコードを追加する**

お使いのドメイン管理サービス（お名前.com、Xserverドメイン、Cloudflareなど）の管理画面で、
以下のDNSレコードを追加します。

- サブドメイン（例: `price.あなたのドメイン.com`）を使う場合:
  - レコード種別: `CNAME`
  - ホスト名: `price`（使いたいサブドメイン名）
  - 値: `ユーザー名.github.io`
- ルートドメイン（例: `あなたのドメイン.com` そのもの）を使う場合:
  - レコード種別: `A`
  - 値: 以下の4つすべてを追加
    ```
    185.199.108.153
    185.199.109.153
    185.199.110.153
    185.199.111.153
    ```

反映まで数分〜最大24時間程度かかることがあります。

**4-2. GitHubリポジトリ側にドメインを設定する**

1. `docs/CNAME` ファイルを開き、中身を実際に使うドメイン
   （例: `price.あなたのドメイン.com`）に書き換えて保存（コミット）
2. リポジトリの「Settings」→「Pages」に戻ると、Custom domain欄に
   自動的にそのドメインが反映されます
3. DNSの反映が完了すると「Enforce HTTPS」にチェックできるようになるので、
   有効にしておくと安全です（httpsで公開されます）

### 5. Yahoo!のClient IDを後日取得したら追加

1. [Yahoo!デベロッパーネットワーク](https://developer.yahoo.co.jp/) にログイン
2. 「アプリケーションの管理」→「新しくアプリケーションを開発」
3. 用途を選び、**Client ID（旧: appid）** を取得（無料・審査なしですぐ発行されます）
4. Secretsに `YAHOO_CLIENT_ID` として追加

> Yahoo未設定の間は、楽天市場の結果のみでランキングが作成されます。

### 6. アフィリエイト収益化を設定する（任意）

リンクをアフィリエイトリンクにしたい場合は、以下2つのASPに登録し、
広告主プログラム（楽天市場／Yahoo!ショッピング）の審査を通してください。

**もしもアフィリエイト（楽天市場向け）**
1. [もしもアフィリエイト](https://af.moshimo.com/) に登録
2. 広告主検索で「楽天市場」のプログラムに提携申請（即時提携が多い）
3. 提携後、発行される `a_id` `p_id` `pc_id` `pl_id` の4つの値を確認
   （管理画面のリンク生成ツールで作られるURLの中に含まれています）

**バリューコマース（Yahoo!ショッピング向け）**
1. [バリューコマース](https://www.valuecommerce.ne.jp/) に登録
2. 広告主検索で「Yahoo!ショッピング」のプログラムに提携申請
3. 提携後、発行される `sid`（サイトID）`pid`（プログラムID）を確認

取得した値をGitHub Secretsに登録します。

| Name | 内容 |
|---|---|
| `MOSHIMO_A_ID` | もしもの a_id |
| `MOSHIMO_P_ID` | もしもの p_id |
| `MOSHIMO_PC_ID` | もしもの pc_id |
| `MOSHIMO_PL_ID` | もしもの pl_id |
| `VALUECOMMERCE_SID` | バリューコマースの sid |
| `VALUECOMMERCE_PID` | バリューコマースの pid |

> 未設定の場合は、通常の商品ページへの直リンクになります（収益化なし）。

サイトのフッターには、景品表示法のステルスマーケティング規制に対応した
「【広告】本サイトはアフィリエイト広告を利用しています。」という表記を
あらかじめ入れてあります。文言は必要に応じて調整してください。

### 7. 手動実行して動作確認

**Actionsタブ → Update lens prices → Run workflow**
で今すぐ実行できます。成功すると `docs/data.json` が更新されるコミットが
自動で作成され、GitHub Pagesに数分以内に反映されます。

以降は毎日 日本時間 朝6時に自動実行されます
（`.github/workflows/update-prices.yml` の `cron` を変更すれば時間は調整できます）。

## サイト名・商品写真・楽天/Yahoo!の表示形式について

- **サイト名**: `scripts/fetch-prices.mjs` の `SITE_NAME`（環境変数 `SITE_NAME` でも上書き可）
  に設定されています。初期値は「ワンデーアキュビューオアシス最安値通販価格情報」です。
- **ランキング表示**: 楽天市場とYahoo!ショッピングを混在させず、
  「楽天市場 最安値TOP5」「Yahoo!ショッピング 最安値TOP5」を別セクションで表示します。
  ページ上部には両方を通じての本日の総合最安値も表示されます。
- **商品画像（ランキング内）**: 楽天・Yahoo!それぞれのAPIが返す商品画像を自動表示します。
  画像が取得できない商品は薄いグレーのプレースホルダーになります。
- **商品写真（ページ上部の3枚）**: `docs/images/product-1.jpg` `product-2.jpg` `product-3.jpg`
  という名前でプレースホルダー画像を用意しています。ご自身で撮影・用意された写真を
  **同じファイル名で上書き**していただければ、そのまま反映されます
  （GitHubの画面から、この3ファイルを同じファイル名でアップロードし直すだけです）。
  枚数を増減したい場合は `docs/index.html` の `product-photos` セクションを編集してください。
- **商品情報セクション（価格ランキングの下）**: 検索対策も兼ねて、商品の特徴を
  説明する文章を掲載しています。文章は `docs/index.html` の `product-info` 内に
  直接書かれているので、自由に書き換えていただけます。
  このセクションの写真は、**楽天商品検索APIが返す実際の商品画像を自動で使用**します
  （毎日の価格更新のたびに、その時点の最安値商品の写真に自動的に更新されます）。
  画像が取得できない場合は `images/product-1.jpg` が代わりに表示されます。

- **冒頭の説明セクション**: 「なぜ90枚入り×2箱セットがお得なのか」という価格比較の
  考え方を、ランキングより先に表示しています。文章は `docs/index.html` の
  `value-explainer` 内に直接書かれているので、自由に書き換えていただけます。
- **口コミ情報セクション（ページ最下部）**: 楽天・Yahoo!それぞれの最安値商品ページへの
  リンクを自動生成します。楽天商品検索API／Yahoo!ショッピングAPIがレビュー件数・
  平均評価を返す場合は、リンクの横に「★4.3・234件のレビュー」のように表示されます。
  レビュー本文そのものは著作権の観点から転載せず、リンクのみをご案内する仕様です。

## 検索キーワード・比較単位を変える場合

- キーワード: `scripts/fetch-prices.mjs` の `KEYWORD` を変更するか、
  環境変数 `SEARCH_KEYWORD` を設定してください。
- 比較単位（何箱セットを対象にするか）: `isTargetBundle()` 関数の正規表現を
  書き換えてください。現在は「90枚入り×2箱（180枚）セット」のみを
  抽出する設定になっています。
- 1箱・2箱・3箱(90枚)など複数の単位を別ページで比較したい場合は、
  `scripts/fetch-prices.mjs` と出力先(`docs/data.json`)をコピーして
  複数セット用意する形が簡単です（参考にされたサイトと同じ構成です。
  ご要望あれば追加対応します）。

## 注意事項

- 本サイトは楽天・Yahoo!の公開APIのみを利用しており、対象サイトの
  スクレイピング（無断でのページ解析・自動取得）は行っていません。
- 表示価格は取得時点のもので、実際の購入時と異なる場合があります。
  免責文言はサイトのフッターに記載済みです。
- `docs/CNAME` はGitHub Pagesの独自ドメイン設定に使われる特別なファイルです。
  誤って削除するとカスタムドメインでのアクセスができなくなるのでご注意ください。
